# 面试答题指导 Skill

## 定位

与 [interview.md](../interview/interview.md) 互补：

| Skill | 视角 | 目标 |
|-------|------|------|
| `interview.md` | 考官 | 怎么出题、怎么评估、怎么归档 |
| `interview-prep.md` | 考生 | 怎么答题、怎么避坑、怎么深挖 |

用户说「答题指导」或询问"面试问到 X 该怎么答"时由本 skill 接管。

## 核心流程

```
用户指定话题（如"智能指针"）
  → 定位 ../../../knowledge-base/fundamentals.md 中对应 `### 子话题`
  → 读取该子话题下全部题目（区分 basic / Advanced）
  → 读取 ../../../data/knowledge-graph.json 中对应节点的出边
  → 按回答分层框架生成指导
```

### 数据源定位规则

1. **话题 → 子话题**：用户在问题中提到的关键词，匹配 `../../../knowledge-base/fundamentals.md` 中各 `## 技术栈` 下的 `### 子话题`
   - 例："智能指针" → `## C++` → `### 智能指针`
   - 例："虚函数" → `## C++` → `### 多态`
   - 支持模糊匹配，取最佳命中
2. **题目提取**：读取该 `###` 节下全部 `- [Stack][Difficulty]` 行
   - `basic` 题 → 基础回答素材
   - `Advanced` 题 → 进阶回答素材
   - 提取该节的 `关联:` 行，作为跨领域提示
3. **图谱查询**：定位 `../../../data/knowledge-graph.json` 中对应 `技术栈:子话题` 节点的出边
   - 每条出边的 `to` 节点 + `label` → 后续追问预测

## 回答分层框架

每个话题输出以下模块：

### 面试官意图
点破这道题真正在考什么。说明考察方向（概念/原理/设计/经验）+ 期望听到的关键信号 + 建议回答时间。

### 精炼回答建议
用最少的话覆盖深度和广度。按"先核心区分、再底层原理、后工程取舍"的主线组织，让面试者能一口气说出有层次且精炼的回答。不需要拆成基础/进阶/深度三层，而是融成一条连贯主线。

### 常见坑
该话题下最容易犯的 2-3 个错误，每条一句话。包括概念混淆、遗漏关键点、过度简化、答非所问等类型。

### 后续追问预测
基于知识图谱出边和 interview.md 的递进策略，预测不同回答深度下考官下一步会怎么问：答了基础会追问什么、答到进阶会深入什么、答出深度可能切到哪个关联领域。

## 使用模式

### 模式一：单题查询
用户明确指定话题，输出该话题的完整答题指导。

> "答题指导：C++ 智能指针"
> "面试问到智能指针该怎么答？"

### 模式二：学习模式
用户指定技术栈未指定子话题，按该栈的全部子话题逐个输出。

> "帮我过一下 C++ 面试准备"
> "面试 C# 一般问什么，怎么答？"

**学习模式下**：
- 参照 [weak-areas.md](../../../weak-areas.md) 优先覆盖薄弱领域
- 每个子话题按顺序逐个输出

## 与 interview.md 评估标准的映射

指导中应引用评估标准来刻度回答质量，让面试者知道"到什么程度拿什么分"：

| 评估 | 考生对应策略 |
|------|-------------|
| 准确 | 覆盖基础+进阶内容，表述精确，能举实际例子 |
| 部分准确 | 基本概念正确，但原理薄弱或有小错误 |
| 不准确 | 核心概念错误或完全不了解 |

## 输出规范

- 使用 Markdown 格式，四个模块用 `###` 标题
- 精炼回答建议不拆多层，按一条主线组织
- 末尾附"关联话题"提示，引导继续学习

## 学习模式话题追踪

学习模式（用户指定技术栈未指定子话题）下，维护 `covered_subtopics` 列表记录已输出的子话题。每完成一个话题输出后更新，避免重复覆盖。优先覆盖 [weak-areas.md](../../../weak-areas.md) 中的薄弱领域。

## 禁止事项

- 禁止直接输出"请使用 pick.py 选题"（那是考官端指令）
- 禁止切换到模拟面试模式（让用户实际答题）
- 禁止生成面试评估或评级
- 禁止修改任何面试数据文件（weak-areas.md、records 等）
