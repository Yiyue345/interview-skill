# AGENTS.md

本文件用于约束 AI Coding Agent 在本项目中的行为。修改、生成或删除代码前，必须优先遵守本文档。

## 核心目标

优先保证项目正确、清晰、安全、可维护、可验证。除非用户明确要求，否则选择最小改动方案。

## 项目约束

- 本项目同时支持 Codex 与 Claude Code。
- Codex Skill 位于 `.agents/skills/`，Claude Code Skill 位于 `.claude/skills/`。
- 修改共享 Skill 时，应同步两套目录；平台专属内容可以不同，但必须说明原因。
- 面试出题必须通过 `src/pick.py`，不得绕过题库直接生成题目。
- 修改题库、索引或知识图谱后，运行 `Build-Knowledge.ps1`。
- 提交前至少运行 `python src/validate_consistency.py`。

## 工作方式

1. 先理解任务目标和影响范围；
2. 先阅读相关代码和文档，再决定修改方案；
3. 优先复用现有结构、工具函数、类型和组件；
4. 只修改与任务直接相关的内容；
5. 修改后说明做了什么、为什么这样做、如何验证。

不要在不了解上下文时直接大段生成代码。需求不清晰且无法从项目中确认时，应询问用户。

## 修改原则

- 保持现有项目风格和公开接口兼容；
- 避免无关重构、格式化、依赖升级和新抽象；
- 避免重复实现已有能力；
- 不删除用户代码，除非任务明确要求；
- 不保留临时代码、调试代码或无用日志；
- 较大改动应先说明范围和理由。

## 安全要求

- 不写入真实 API Key、Token、密码、私钥、Cookie、数据库连接串或个人隐私信息；
- 使用占位符、环境变量或配置示例表达敏感配置；
- 涉及输入、文件、命令、数据库和网络时，考虑注入、权限绕过、路径穿越、非法输入、超长输入、空值、重复提交和异常失败；
- 未经明确要求，不执行破坏性数据操作、Git 历史重置、强制推送、生产配置修改或不可逆数据库操作。

## Bug 与重构

修复 Bug 时尽量说明原因、修改位置、修复思路、兼容性影响和验证方式。无法确定根因时，应明确当前判断和不确定点。

重构必须服务于明确目标，默认不改变外部行为；如果行为发生变化，必须明确说明。

## 测试与文档

- 优先使用项目已有的单元测试、集成测试、类型检查、lint、构建或本地运行方式；
- 不声称运行过未实际运行的验证；
- 无法运行验证时，说明原因和建议检查步骤；
- 修改使用方式、配置、数据结构、启动或部署流程时，同步更新相关文档。
